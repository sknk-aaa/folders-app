# ブックマークアプリ 開発仕様書

> Claude Code向け開発指示書 / iOS向けブックマーク管理アプリ

---

## 1. プロジェクト概要

### 1.1 アプリの目的
スマホのブラウザから手軽にブックマークを保存・管理できる、見た目重視のブックマークアプリ。サイトのサムネイル画像を自分で選択でき、視覚的にブックマークを整理できることを最大の特徴とする。

### 1.2 ターゲットユーザー
- スマホでブラウジングをしながら気になったページを保存したい人
- ブラウザ標準のブックマーク機能では視覚的に分かりづらいと感じている人
- フォルダ分けして情報を整理したい人

### 1.3 配布形態
- iOS版のみ（App Store配布）
- 開発者：個人開発（Apple Developer登録必要、年99ドル）

## 2. 技術スタック

### 2.1 開発環境
- **OS**: Windows（Macは不要）
- **エディタ**: 任意
- **実装**: Claude Code

### 2.2 フレームワーク・ライブラリ
| 項目 | 採用技術 | 備考 |
|---|---|---|
| フレームワーク | Expo (Bare Workflow) | React Nativeベース |
| ビルド | EAS Build | クラウドビルド、Windows対応 |
| 言語 | TypeScript | 型安全のため推奨 |
| 状態管理 | 任意（Zustand推奨） | アプリ規模的に軽量なものでOK |
| ローカルDB | SQLite (expo-sqlite) | ブックマーク・フォルダ保存 |
| 画像処理 | expo-image-manipulator | サムネトリミング |
| 共有受け取り | Share Extension（ネイティブ実装） | iOS共有メニュー対応 |
| 広告 | Google AdMob (react-native-google-mobile-ads) | バナー広告 |
| 課金 | react-native-iap | 買い切り課金 |
| バックアップ | iCloud / Google Drive API | プレミアム機能 |

### 2.3 Bare Workflowを採用する理由
Share Extension（iOS共有メニューからの受け取り）を実装するため、ネイティブコード（Swift/Objective-C）の追加が必要。Managed Workflowでは対応不可。

### 5.1 共通UI（常時固定ヘッダー）

すべての画面で上部に以下を表示：

### 5.2 タブ切替

ヘッダー下に2つのタブを配置：
- **フォルダー**（デフォルト選択）
- **ブックマーク**

タブ切替時はアニメーション付き（スライド）。

### 5.3 フォルダー一覧画面

#### 5.3.1 レイアウト
```
┌─────────────────────────────┐
│ [☰] アプリ名 [🔍][＋]        │  ← 固定ヘッダー
├─────────────────────────────┤
│ [フォルダー]  ブックマーク    │  ← タブ
├─────────────────────────────┤
│  ┌──┐  ┌──┐               │
│  │📄│  │🎵│               │  ← フォルダグリッド（2列固定）
│  │記事│ │音楽│              │
│  └──┘  └──┘               │
│  ┌──┐  ┌──┐               │
│  │💼│  │🛒│               │
│  │仕事│ │買物│              │
│  └──┘  └──┘               │
│                              │
├─────────────────────────────┤
│ 📌 最近追加したブックマーク    │  ← セクション見出し
│  [サムネ][サムネ][サムネ]→   │  ← 横スクロール
├─────────────────────────────┤
│ [広告バナー]（無料版のみ）    │
└─────────────────────────────┘
```

#### 5.3.2 フォルダグリッド
- 2列固定（列数変更不可、ブックマークのみ拡縮対応）
- 各セルに「アイコン」「フォルダ名」を表示
- タップでフォルダ詳細画面に遷移
- 長押し → ホールド＆スライドで並び替え可能

#### 5.3.3 最近追加したブックマーク
- 最新10件を横スクロールで表示
- サムネ画像（正方形）＋サイト名
- タップでサイトに飛ぶ（デフォルトブラウザで開く）
- 初回起動時はGoogle（https://google.com）を「未分類」に1件追加しておく

#### 5.3.4 広告バナー
- フォルダー一覧画面の最下部に固定
- 無料版のみ表示、有料版では非表示
- AdMobの標準バナーサイズ（320×50 もしくは Adaptive Banner）

---

### 5.4 ブックマーク一覧画面（全ブックマーク）

#### 5.4.1 仕様
- 全ブックマークを **追加順（新しい順）固定** で表示
- 並び替え機能なし
- 表示モード（グリッド/リスト）切替可能
- グリッド時の列数は変更可能（2本指ピンチ）
- フォルダフィルタ機能なし
- 広告バナーは非表示

#### 5.4.2 表示モード
**グリッド表示（初期は2列、ピンチで2/3/4列切替）：**
```
┌─────────────────────┐
│ [favicon] サイト名 [⋮]│
│ ┌─────────────────┐ │
│ │                  │ │
│ │   サムネ画像     │ │  ← 正方形
│ │                  │ │
│ └─────────────────┘ │
└─────────────────────┘
```

**リスト表示：**
```
┌─────────────────────────────┐
│ [□] サイト名               [⋮]│  ← □は小さいサムネ正方形
└─────────────────────────────┘
```
- 左：サムネ小（正方形）
- 右：サイト名のみ（URLは表示しない）
- 右端：⋮メニュー

#### 5.4.3 共通操作
- **タップ**：デフォルトブラウザでサイトを開く
- **⋮メニュー**：「削除」「移動」「編集（名前・URL変更）」
- **長押し→スライド**：並び替え・フォルダ移動
  - ※ブックマーク一覧では並び替え不可（追加順固定）。フォルダ詳細画面でのみ並び替え可能。

#### 5.4.4 画像読み込み失敗時
- 「画像がありません」と表示されたプレースホルダ画像を出す
- グレー背景＋中央にアイコン＋テキストのシンプルな画像

---

### 5.5 フォルダ詳細画面

#### 5.5.1 仕様
- 選択したフォルダ内のブックマークのみ表示
- ヘッダー左上：ハンバーガー → 「＜戻る」に変化
- 表示モード・列数はブックマーク一覧と同じ
- このフォルダ内に新規追加する場合、＋ボタンから追加すると自動でこのフォルダに入る

#### 5.5.2 並び替え
- ホールド＆スライドで並び替え可能
- ホールド＆スライドで別フォルダにドラッグ → フォルダ移動

#### 5.5.3 フォルダ自体の操作
- フォルダー一覧画面でフォルダの⋮メニュー（長押し or 何らかのアクセス手段）から「編集」「削除」
  - **要設計判断**：フォルダグリッドには⋮がないため、長押しでメニュー出現とするか、編集モードを設けるか実装時に検討
  - **推奨案**：フォルダを長押しで「並び替え or メニュー」の選択UIを出す。または編集モードボタンを設ける
- **編集**：フォルダ名・アイコン変更
- **削除**：確認ダイアログ → 中のブックマークも全て削除

### 5.6 ブックマーク追加フロー

#### 5.6.1 起動経路
1. **シェアエクステンション経由**：ブラウザ（Safari/Chrome/Edge）の共有ボタン → 「その他」→ 本アプリを選択 → バックグラウンドで処理開始 → 必要なら前面表示
2. **アプリ内＋ボタン経由**：現在のフォルダ（もしくは未分類）に追加

#### 5.6.2 追加フロー（サムネ撮影ON時）
```
1. 共有から起動 / ＋ボタン押下
   ↓
2. URLを取得・サイトのスクリーンショットを取得
   （バックグラウンドでWebViewレンダリング、もしくはOGP画像取得）
   ↓
3. 全画面でスクリーンショットを表示、トリミング画面に遷移
   ↓
4. ユーザーがトリミング範囲を指定（ドラッグでリサイズ可能な矩形）
   ↓
5. 「次へ」タップ
   ↓
6. メタ情報入力画面：
   - サイト名（自動取得、編集可能）
   - URL（自動取得、編集可能）
   - 保存先フォルダ（デフォルト：未分類 or 直近選択フォルダ）
   ↓
7. 「保存」タップ
   ↓
8. トースト「ブックマークを追加しました」
   ↓
9. シェア経由：そのままブラウザに戻る
   アプリ内：元の画面に戻る
```

#### 5.6.3 追加フロー（サムネ撮影OFF時）
- ステップ3〜5（トリミング画面）をスキップ
- サイトトップページのスクリーンショットを自動取得してサムネ化
- 取得失敗時は「画像がありません」プレースホルダ

#### 5.6.4 同URLの重複登録
- 警告は出さず、そのまま追加（重複OK）

#### 5.6.5 100件上限の挙動（無料版）
- 90件目で警告トースト：「残り10件です（プレミアムで無制限に）」
- 100件達成後、追加しようとしたらトースト：「ブックマーク上限に達しました。プレミアム版で無制限になります」
- 上限解放は買い切り課金で

### 5.7 検索機能

#### 5.7.1 仕様
- 右上の🔍アイコンから検索画面起動
- **フォルダ詳細画面内でのみ動作**（全ブックマーク横断検索ではない）
- 検索対象：サイト名 + URL
- リアルタイム絞り込み（入力1文字ごとにフィルタ更新）
- 検索結果は通常の表示モード（グリッド/リスト）で表示
- フォルダ名は結果に表示しない

#### 5.7.2 UI
- 検索バーが上部に出現、キーボード自動表示
- 検索結果がリアルタイムで下に表示
- 「キャンセル」ボタンで検索終了

## 6. 収益モデル

### 6.1 無料版
- 広告あり（フォルダー一覧画面の下部バナーのみ）
- ブックマーク登録上限：**100件**
- フォルダ数上限：なし

### 6.2 プレミアム版（買い切り480円）
- 広告非表示
- ブックマーク無制限
- iCloud / Google Driveバックアップ機能

### 6.3 課金実装
- `react-native-iap` を使用
- App Storeの「Non-Consumable」プロダクトとして登録
- プロダクトID例：`com.yourname.bookmarkapp.premium`
- 購入状態は端末ローカルとApple IDの両方で管理
- 「購入を復元」機能を必ず実装（App Store審査要件）

## 7. iOS Share Extension 実装

### 7.1 概要
Safari/Chrome/Edgeの共有ボタンから本アプリを起動できるようにする。

### 7.2 実装方法
- `expo-share-extension` または独自ネイティブ実装
- Xcodeプロジェクトに Share Extension target を追加（EAS Buildで自動化）
- `Info.plist` に NSExtension 設定を追加
- URLとタイトルを受け取り、メインアプリのURLスキーム経由でデータを渡す or App Group経由でデータ共有

### 7.3 受け取るデータ
- ページURL
- ページタイトル
- （可能なら）スクリーンショット or favicon URL

### 7.4 動作フロー
1. ブラウザで共有ボタン → アプリ選択
2. Share Extensionが起動（軽量ビュー）
3. URLとタイトルを取得し、App Group経由でメインアプリに渡す
4. メインアプリが前面起動、ブックマーク追加フローに遷移
5. ※完全バックグラウンド処理は不可（ユーザーが何らかの確認をする画面が必要）

## 10. 開発フェーズ提案

### Phase 1: コアアプリの完成（MVPまで）
1. プロジェクトセットアップ（Expo Bare Workflow + TypeScript + SQLite）
2. データ層実装（フォルダ・ブックマークのCRUD）
3. フォルダ一覧画面 / ブックマーク一覧画面 / フォルダ詳細画面
4. ブックマーク追加フロー（アプリ内のみ、サムネトリミング含む）
5. ⋮メニュー・編集・削除
6. ホールド＆スライド並び替え・フォルダ移動

### Phase 2: 共有エクステンション統合
1. Share Extension のネイティブ実装

## 11. 注意事項・Claude Codeへの指示

### 11.1 開発環境
- **Windowsで開発する前提**で進めてください
- iOSビルドはすべて **EAS Build** を使用
- Xcodeが必要な作業はEAS Buildに任せる

### 11.2 コーディング規約
- TypeScriptで書く
- 関数コンポーネント + Hooks
- ESLint + Prettier導入
- ファイル構成は機能ベースで分割（`/features/folders`, `/features/bookmarks` など）

### 11.3 動作確認
- Expo Goでの確認は不可（Bare Workflowのため）
- EAS Build → TestFlightで実機確認

### 11.4 不明点があった場合
仕様が曖昧な箇所は実装前に質問すること。特に以下の領域は実装時に判断が分かれる可能性が高い：
- フォルダの長押し動作（並び替え or メニュー）
- サイトのスクリーンショット取得方法（WebView/OGP/サードパーティAPI）
- Share Extensionのデータ受け渡し方式
